# Helix — спецификация переписывания на современный React

> Рабочий документ для миграции прототипа (HTML + React UMD + Babel Standalone)
> на Vite + React 18 + ES modules. Ветка: `react-rewrite`.
> Эталонные скриншоты прототипа: `docs/screenshots-baseline/` (21 шт., сняты 2026-06-09 с master).
> Полное продуктовое видение — `REQUIREMENTS.md`, правила проекта — `CLAUDE.md`.

## Цель

Переписать кодовую базу как настоящее React-приложение **без потери функционала и дизайна**.
Поведение и внешний вид должны совпасть с эталонными скриншотами 1-в-1
(включая мелкие странности прототипа — см. «Поведение, которое сохраняем как есть»).

## Целевой стек

- **Vite + @vitejs/plugin-react**, plain JS/JSX (без TypeScript — минимизируем риск регрессий в теории).
- React 18 + react-dom из npm (вместо UMD CDN).
- `soundfont-player` из npm (вместо CDN unpkg); сэмплы MusyngKite грузятся как раньше с gleitz/midi-js-soundfonts.
- CSS — обычные файлы, импортируются модулями-владельцами (`import './instrument.css'`).
- Никаких `window.*`-регистраций: только ES-импорты/экспорты.

## Требования к качеству кода (от пользователя)

1. **Модули по принципу владения функционалом**: каждый модуль владеет своей областью
   (теория, аудио, инструмент, круг, поиск, диаграммы, закреплённые, топбар, tweaks).
   CSS режется по тем же границам (сейчас `circle.css` стилизует и круг, и диаграммы, и поиск — разделить).
2. **React-компоненты — не более 4 уровней отступа** внутри функции компонента.
   Практически: глубокий JSX выносится в подкомпоненты, колбэки map — в отдельные
   компоненты/функции, ранние return вместо вложенных условий.

Правила для агентов:

- Импорт теории — `import { scaleNotes, MODES, ... } from '<путь>/theory'` (через index.js).
- Аудио — `import * as HelixAudio from '<путь>/audio'` либо именованные импорты; сигнатуры НЕ менять.
- `ROMAN_MAJOR` / `ROMAN_MINOR` — легаси-алиасы; перед удалением проверить, что не используются.
- Деструктуризацию `const { useState } = React` заменить на нормальные импорты из 'react'.
- Логику не «улучшать» по дороге. Музыкальная теория критична (см. CLAUDE.md):
  любые изменения qualities/энгармоники/интервалов запрещены — только перенос.
- Скоринг генератора войсингов переносить дословно.

### Центр
- [ ] Шапка: «ГИТАРА» / «БАС ГИТАРА (5)» / «ФОРТЕПИАНО»; чип активного аккорда «Dm ×»
      (клик по × снимает выделение); справа «Подсветка: ступени».
- [ ] Гриф: 15/18 ладов, номера ладов, инлеи 3/5/7/9/15 + двойной 12, нут, выделенный октавный лад.
      Ноты гаммы — цветные кружки с номером ступени; цвет = ступень тональности.
- [ ] При активном аккорде гриф показывает только хорд-тоны: корень «R», остальные — номера
      ступеней тональности, цвета ступеней.
- [ ] Клик по ноте: pulse + сэмпл (питч с учётом лада/строя/каподастра).
- [ ] Фортепиано: 3 октавы от C3 (подписи C3/C4/C5), та же подсветка, клик играет ноту.
- [ ] Квинтово-квартовый круг (SVG): внешнее кольцо мажоры (C сверху, по квинтам), внутреннее —

## Поведение, которое сохраняем как есть (не чинить молча)

Замечено в прототипе; это НЕ блокеры, переносим поведение 1-в-1, чтобы ничего не сломать.
Чинить — отдельным решением пользователя после миграции:

1. Чип активного аккорда всегда в красной тонировке `--deg-1`, независимо от ступени.
2. «По позиции» — заглушка (identifyChord в теории готов, но не подключён).
3. При активном «Поиске по имени» с непустым запросом результаты имени показываются, даже если
   подсвечен таб «По нотам»/«По позиции».
4. Escape не закрывает дропдаун тональности.
5. Свёрнутый круг прячет и «Аккорды тональности».
6. favicon отсутствует (404 в консоли) — можно добавить пустой, это не изменение дизайна.

## Дизайн-токены (перенести дословно из styles.css)

- Светлая: `--bg #ffffff`, `--bg-muted #f7f7f5`, `--text #1f1f1d`, бордеры rgba(15,15,14,…), мягкие тени.
- Тёмная: `--bg #0e0e0d`, `--bg-elev #1a1a18`, ступени L 0.72, тонировки L≈0.28.
- Ступени 1–7: OKLCH L0.62 C0.18, hues 25/60/110/155/230/290/340 + `-bg`-тинты (L0.96 C0.04).
- `--secondary` (втор. доминанты), `--in-key` (зелёный). Радиусы 4/6/10/14, пилюли 999px.
- Шрифты: Inter (UI), JetBrains Mono (ноты/аккорды/римские/бейджи). Google Fonts остаются в index.html.

## Ключевые данные (не потерять при переносе)

- SCALES: 11 ладов; MODES с qualities/romans (null у пентатоник); CHORD_TYPES — 19 типов;
  CIRCLE_MAJOR/MINOR; TUNINGS (guitar6 — 6 пресетов, bass5 — 3).
- chord-shapes-data.js: ~300 курированных гитарных форм из chords-db (MIT),
  записи `{ baseFret, positions[{string,fret,finger}], bassNote }`, ключи вида `C`, `Cm7`, `C/E`.
- Дефолтные закреплённые: C, G, Am, F.
- Скоринг генератора: `Math.max(0, lowFret-3)*1 + span*1.5 + internalMutes*4 + бонус полноты (≥5 струн)`.

## Известные ловушки миграции

- Убрать ВСЕ `window.*` (MT, HelixAudio, CHORD_SHAPES_DB, flashPulse, компоненты) → ES-модули.
  flashPulse дёргается из многих мест — сделать `src/ui/pulse.js` и импортировать.
- soundfont-player: npm-пакет `soundfont-player@0.12.0`, `import Soundfont from 'soundfont-player'`.
  Дефолтный CDN сэмплов внутри пакета — тот же gleitz GitHub; проверить, что URL не изменился.
- chord-shapes-data: перегенерировать как ES-модуль (`node scripts/build-chord-shapes.js` после правки скрипта)
  или механически заменить `window.CHORD_SHAPES_DB =` на `export default`.
- OKLCH требует свежих браузеров — оставить как есть (прототип уже использует).
- AudioContext warning до первого жеста — норма, не «чинить» созданием по клику, если не просто.
- SRI-хэши для CDN не возвращать (их больше и не будет — всё из npm).
- index.html переписывается с нуля под Vite: `<div id="root">` + `<script type="module" src="/src/main.jsx">`,
  Google Fonts и favicon — в head.

## Как проверять (приёмка)

Прогнать сценарии браузерного отчёта (см. имена скриншотов в docs/screenshots-baseline/):
initial → key dropdown → смена тональности слева → пентатоника → клик ноты → активный аккорд →
piano/bass/DADGAD+capo → круг (клик сектора, сворачивание) → поиск Cmaj7 → конструктор C maj →
«По позиции» → пины → аккорд/арпеджио + mute → тёмная тема → скрытие панелей → узкий viewport
(меню/поиск drawer-ы, бекдроп). Консоль должна быть чистой (кроме AudioContext warning до жеста).
